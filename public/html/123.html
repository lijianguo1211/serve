## **什么是依赖注入** 【依赖注入】通过以构造函数参数，设值方法或属性字段等方式将具体组件传递给依赖方。 这就是一句解释，我们一看貌似更加糊涂了。这到底什么跟什么呀。 在PHP的框架laravel里面，就用到了它 **IOC（Inversion of Control）**模式，它又称依赖注入（Depe-ndency Injection）模式。也就是说它就是一种设计模式，一种思想。 这里有两种说法 - 控制反转是将组件间的依赖关系从程序内部提到外部容器来管理 - 依赖注入是指组件的依赖通过外部参数或其它形式注入 - 依赖 ： 就是基类，Interface (接口), Abstract (抽象) - 注入 ： 就是一个 new 的过程，可能有容器模式实例化，可能其他模式。 下面我们在具体的实现之前先看点别的小例子 我们现在有一个动物类【Animal.php】,这个动物类呢要通过继承别的类【就是具体的动物】才能有自己的特点 ```php _animal = $animal; } } ``` 下面是一个具体的动物类【Species.php】 Animal类通过继承Species类才能让自己变成小奶狗 ```php _Species = $Species; } } ``` 每只动物都应该有自己的名字，性别，年龄，这个小奶狗也是一样的，所以它也需要继承【Introduce.php】来做自我介绍 ```php _introduce = $introduce; } } ``` 我们现在想让一个小动物做自我介绍的话，我们就需要一个的继承，这样用起来很麻烦
